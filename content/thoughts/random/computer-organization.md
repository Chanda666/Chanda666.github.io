# 组成原理

记录计算机组成原理中的 CPU、Cache、流水线、存储系统、指令执行流程等内容。

## Cache 与存储层次

高速缓存（Cache）是位于 CPU 和主存之间的高速小容量存储器，用于解决 CPU 与主存之间的速度不匹配问题。

### 存储层次结构

```
寄存器 → L1 Cache → L2 Cache → L3 Cache → 主存 → 磁盘
   1ns      2ns       10ns       30ns     100ns   10ms
```

### Cache 映射方式

| 映射方式 | 特点 | 冲突概率 |
|---------|------|---------|
| 直接映射 | 每个主存块固定映射到一个 Cache 行 | 高 |
| 全相联映射 | 主存块可映射到任意 Cache 行 | 低（但硬件复杂） |
| 组相联映射 | 折中方案，每组内全相联 | 中等 |

### 替换策略

- **LRU**（Least Recently Used）：替换最久未使用的块
- **FIFO**（First In First Out）：替换最早进入的块
- **随机替换**：随机选择一个块替换

## 流水线 CPU

流水线技术将指令执行过程划分为多个阶段，使多条指令可以重叠执行。

### 经典五级流水线

1. **IF**（Instruction Fetch）：取指令
2. **ID**（Instruction Decode）：译码 / 读寄存器
3. **EX**（Execute）：执行 / 计算地址
4. **MEM**（Memory Access）：访问数据存储器
5. **WB**（Write Back）：写回寄存器

### 流水线冒险

| 类型 | 描述 | 解决方案 |
|------|------|---------|
| 结构冒险 | 硬件资源冲突 | 增加硬件资源 |
| 数据冒险 | 指令间数据依赖 | 前推（Forwarding）、阻塞（Stall） |
| 控制冒险 | 分支指令导致 | 分支预测、延迟槽 |

```
# 数据冒险示例
add x1, x2, x3    # x1 = x2 + x3
sub x4, x1, x5    # 需要 x1 的值，但 add 还未写回
```

## 指令执行流程

### RISC-V 指令格式

RISC-V 有六种基本指令格式：

- **R-type**：寄存器-寄存器操作（add, sub, and, or）
- **I-type**：带立即数的操作（addi, lw, jalr）
- **S-type**：存储指令（sw）
- **B-type**：分支指令（beq, bne）
- **U-type**：长立即数（lui, auipc）
- **J-type**：跳转指令（jal）

### 单周期 vs 多周期 vs 流水线

| 实现方式 | CPI | 时钟周期 | 特点 |
|---------|-----|---------|------|
| 单周期 | 1 | 最长指令延迟 | 简单但慢 |
| 多周期 | 3-5 | 最慢阶段延迟 | 不同指令不同周期数 |
| 流水线 | ~1 | 最慢阶段延迟 | 吞吐率高 |

## 中断与异常

- **异常（Exception）**：由指令执行引起的同步事件（如除零、非法指令）
- **中断（Interrupt）**：由外部设备引起的异步事件（如 I/O 请求）

### 处理流程

1. 保存当前 PC（程序计数器）
2. 根据异常/中断类型查询异常向量表
3. 跳转到对应的处理程序
4. 处理完毕后恢复执行（或终止）
